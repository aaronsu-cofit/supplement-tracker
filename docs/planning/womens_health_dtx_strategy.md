# 女性身心健康 DTx 產品策略與執行規劃

> **版本：** v1.0
> **日期：** 2026-03-11
> **載體：** LINE Official Account + LIFF
> **定位：** 給 35-55 歲女性的身心健康陪跑服務

---

## 一、背景與策略邏輯

### 為什麼選擇這個方向

Cofit 在減脂體重管理領域建立了成熟的 DTx 模式（數位處方簽），核心能力是：
- 透過飲食與生活型態改變幫助使用者達成健康目標
- 以「陪跑」形式提供持續性支持
- 整合任務模組、知識內容、互動訊息的完整服務流程

老闆提出的 **A+B+C+D 框架**（A 線上課程、B 任務模組、C AI 工具、D 訊息模板）正是這套模式的抽象化，目標是複製到其他醫療領域。

**選擇「女性身心健康」作為優先擴張領域的理由：**

| 評估維度 | 說明 |
|---------|------|
| 與減脂模式相似度高 | 同樣是生活型態長期改變，飲食、運動、情緒管理高度重疊 |
| 長期陪伴需求強 | 更年期是 5-10 年的身體轉變，不是一次性解決問題 |
| 目標族群付費意願高 | 35-55 歲女性有消費能力，且問題嚴重影響生活品質 |
| 醫療人力缺口明顯 | 婦產科診所無人力做出院後或門診外的長期追蹤 |
| 競爭市場尚未成熟 | 台灣女性健康數位服務極度缺乏 |
| 無需診所關係可啟動 | 性治療與更年期照護使用者有強烈自主搜尋動機，可 B2C 切入 |

## 二、產品定位

### 名稱與定位建議

> **「Vitera 女性身心健康陪跑」**
> 從更年期症狀管理到親密關係健康，一個懂你的數位陪伴

**不說：**「更年期 App」、「性治療平台」（污名化、高門檻）

**說：** 「妳的身體在轉變，我們陪妳一起走過」

---

## 三、目標族群

### 主要族群：35-55 歲女性

細分為三種進入樣態：

| 樣態 | 主要需求 | 進入觸發點 |
|------|---------|----------|
| **更年期過渡期** | 症狀管理（熱潮紅、睡眠、情緒、體重）| 搜尋熱潮紅、睡不好、更年期 |
| **親密關係困擾** | 低慾望、疼痛、乾澀、與伴侶疏離 | 搜尋私密不適、夫妻關係 |
| **身心狀態下滑** | 體力不佳、情緒失調、不知道原因 | 減脂服務自然延伸 |

## 四、A+B+C+D 框架詳細設計

### A — 線上課程與影片

按症狀組合動態推送，非所有人看同樣內容。

**核心課程模組（每模組約 2 週）：**

| 代號 | 主題 | 觸發條件 |
|------|------|---------|
| M1 | 認識更年期：身體在發生什麼 | 所有人必看（入門） |
| M2 | 飲食調整：植物雌激素、鈣質、抗發炎 | 熱潮紅、體重上升 |
| M3 | 骨盆底肌訓練（凱格爾）完整教學 | 漏尿、性交疼痛、低慾望 |
| M4 | 睡眠修復計畫 | 睡眠評分 < 3 |
| M5 | 情緒與壓力管理 | 情緒波動、焦慮評分高 |
| M6 | 親密關係重建 | 低慾望、夫妻疏離 |
| M7 | 骨質疏鬆預防與運動 | 45 歲以上、BMI 偏低 |

**影片格式規範：**
- 長度：3-5 分鐘短片（行動裝置最佳瀏覽體驗）
- 風格：溫暖、專業、去醫療化語言
- 推送方式：LINE 訊息卡片預覽 → 點擊開 LIFF 播放

---

### B — 任務模組

每日透過 LINE 訊息推送，點擊後開 LIFF 完成打卡。

**更年期症狀管理方案（每日任務）：**

```
✅ 今日症狀記錄
   - 熱潮紅次數（0-10 次）
   - 睡眠品質評分（1-5 分）
   - 情緒狀態（Emoji 選擇）

✅ 骨盆底肌訓練 x3 組
   - LIFF 內計時器引導（收縮 5 秒 / 放鬆 10 秒）
   - 完成記錄連續天數

✅ 飲食打卡
   - 今天有沒有吃到建議食物？（豆製品、深綠色蔬菜、鮭魚）

✅ 睡前放鬆任務（選做）
   - 呼吸練習引導 / 伸展動作
```

**親密關係改善方案（每日任務）：**

```
✅ 今日身體感受記錄（私密日記，僅自己看得到）
✅ 骨盆底肌訓練（同上）
✅ 本週感官練習（Sensate Focus，搭配影片引導）
✅ 今日與伴侶互動打卡（可選擇性，非強制）
```

**週任務：**
- 每週日推送「本週成就回顧」+ 下週預告
- 依完成率給予不同的鼓勵訊息

---

### C — AI 賦能工具（LIFF 核心功能）

**工具 1：初始症狀評估與方案分流**

> 這是最重要的功能，也是免費開放的流量入口。

流程：
1. 10-15 題問卷（症狀頻率、嚴重度、生活影響）
2. 使用滑動條或 Emoji 程度評分（降低填答壓力）
3. 輸出：症狀雷達圖 + 個人化方案推薦 + 最相關的 Top 3 課程

評估維度：
- 更年期症狀（熱潮紅、睡眠、情緒、骨骼）
- 性功能指標（慾望、舒適度、滿意度）
- 生活型態（壓力、睡眠、飲食、運動）

---

**工具 2：症狀日記分析（每 2 週自動生成）**

> 讓使用者感受到「有人在看我的數據」。

輸出範例：
```
📊 妳的雙週分析報告

熱潮紅：本期平均 3.2 次/天，較上期減少 15% ✅
睡眠品質：平均 2.8 分，仍偏低 ⚠️
骨盆底肌訓練：完成率 71%，連續最長 8 天 🎉

💡 AI 觀察：
妳的熱潮紅在週三、週四特別嚴重，
與這兩天的壓力評分偏高有相關性。
建議嘗試這週的「壓力管理課程 M5」。
```

---

**工具 3：骨盆底肌訓練引導器**

> 核心差異化功能，同時適用更年期與性健康。

功能規格：
- 視覺化收縮/放鬆節奏引導（進度條動畫）
- 三種訓練模式：初級（5 秒）/ 進階（10 秒）/ 快速收縮
- 每日完成記錄 + 連續天數顯示
- 30 天前後比較說明（為什麼持續很重要）

---

**工具 4：AI 私密問答**

> 使用者最不敢問醫師的問題，在這裡可以匿名問。

設計原則：
- 入口文案：「這裡是妳的私密空間，沒有評判，只有理解」
- 以 Claude API 實作，System Prompt 強調：同理語氣、有醫療依據、不給診斷建議、適時建議就醫
- 常見問題預設快問快答（降低輸入門檻）
- 聊天記錄僅本人可見

---

### D — 訊息模板

LINE OA 推送的訊息要讓人覺得「有溫度」，不像廣告。

**每日推送節奏：**

| 時間 | 訊息類型 | 範例 |
|------|---------|------|
| 08:00 | 每日任務推送 | 「早安 ☀️ 今天有三個小任務等著妳，昨天完成了 2/3，今天繼續！→ [開始今日任務]」 |
| 21:00 | 症狀記錄提醒（依狀況觸發）| 「記得記錄一下今天的狀況嗎？妳的記錄幫助我們更了解妳的身體 🌙 → [記錄今日症狀]」 |

**特殊觸發模板：**

| 觸發條件 | 訊息策略 |
|---------|---------|
| 連續 3 天未打卡 | 關心而非催促：「這幾天還好嗎？不用勉強，妳隨時回來都可以 💙」 |
| 完成第一週 | 成就慶祝 + 解鎖新課程通知 |
| 症狀記錄顯示本週特別辛苦 | 鼓勵訊息 + 推送最相關的影片 |
| 連續訓練 7 天 | 里程碑慶祝 + 成效說明（為什麼第 7 天很重要）|
| 月經週期接近（如有記錄）| 提醒症狀可能加劇 + 預防性建議 |

## 六、技術架構（LINE OA + LIFF）

```
LINE OA
├── 訊息推送（Messaging API）
│   ├── 每日任務卡片
│   ├── 特殊觸發訊息
│   └── 課程解鎖通知
│
└── LIFF 應用（Next.js，整合 LINE Login）
    ├── /onboarding  → 初始評估問卷 + 方案分流
    ├── /dashboard   → 使用者主頁（今日任務、進度、連續天數）
    ├── /tasks       → 每日任務打卡
    ├── /kegel       → 骨盆底肌訓練引導器
    ├── /courses     → 課程影片列表與播放
    ├── /report      → 雙週症狀分析報告
    ├── /ai-chat     → AI 私密問答
    └── /shop        → 保健品推薦（串接既有商城）
```

**認證：** LINE Login → 取得 userId → 後端建立使用者 profile
**資料存儲：** 每日打卡、症狀記錄、訓練記錄、問卷結果
**AI：** Claude API（問答）+ 後端分析邏輯（症狀報告生成）

## 七、商業模式

### 定價結構

| 方案 | 內容 | 建議定價 |
|------|------|---------|
| 免費體驗 | 初始評估 + 前 2 週任務模組 + 2 支課程影片 | 免費 |
| 月費訂閱 | 完整任務模組 + 全部課程 + 雙週報告 | NT$299-399/月 |
| 季費訂閱 | 月費內容 + AI 問答 + 優先客服 | NT$799-999/季 |
| 諮詢加購 | 一對一視訊（30 分鐘，與合作護理師/諮商師）| NT$800-1,200/次 |
| 保健品方案 | AI 推薦搭配 + 訂閱優惠 + 提醒服用 | 依品項 |

### 保健品整合策略

與其獨立販售保健品，應整合進方案：
- **症狀分析報告中自然推薦**：「妳的鈣質攝取不足評分偏高，這是我們推薦的補充方案」
- **服用提醒綁定任務模組**：設定提醒後計入每日打卡完成率
- **成效追蹤**：「連續服用 4 週後，妳的骨骼相關症狀分數提升了 X 分」

## 八、MVP 執行計畫

目標：驗證使用者願意每天打開 LINE 完成任務

**必做功能：**
- [ ] LINE OA 設定 + 訊息推送基礎流程
- [ ] LIFF 初始評估問卷（工具 C1）
- [ ] 每日症狀記錄（任務 B）
- [ ] 基礎每日任務推送訊息模板（D）
- [ ] 課程影片 M1、M3（2 支先上）
- [ ] 骨盆底肌訓練引導器（C3）

**成功指標：**
- D7 留存率 > 40%
- 每日打卡完成率 > 50%
- 問卷完成率 > 80%

**必做功能：**

## 九、與既有 Vitera 架構的關係

本服務可作為獨立的 `intimacy` 或新的 `womens-health` app 存在於 Vitera monorepo 中，共用：

- `@vitera/lib` 的認證與 API fetch 機制
- 既有的後端 API 架構
- 保健品商城資料

已有的 `apps/intimacy` 功能規劃文件（`sexual_health_planning.md`）聚焦在性健康功能細節，本文件作為上層策略與完整服務藍圖，兩份文件互為補充。

## 十、待討論與決策事項

| 問題 | 選項 | 建議 |
|------|------|------|
| 性健康與更年期是同一個入口還是分開？ | A 合併為女性健康 / B 分開兩個 OA | 建議合併，降低使用者標籤化心理 |
| 付費機制用哪個？ | LINE Pay / 信用卡 / 第三方金流 | LINE Pay 體驗最順暢 |
| AI 問答的免責聲明如何處理？ | 顯示警語 / 僅建議就醫 | 每次回答結尾附「此為健康資訊，非醫療診斷」 |
| 初期冷啟動怎麼找到第一批使用者？ | Cofit 現有用戶轉化 / 社群投放 | 優先從現有 Cofit 女性用戶轉化 |
| 是否需要合作的婦科護理師或治療師背書？ | 是 / 否 | 建議至少找 1-2 位掛名顧問，增加可信度 |
