# A+B+C+D 框架功能規格

> **版本：** v1.1
> **日期：** 2026-03-12
> **說明：** 本文件定義 A+B+C+D 四個元件的通用功能與各場景的專屬功能，適用於所有 Vitera DTx 產品線。
> **狀態更新：** A（線上課程平台）已完成開發，MVP 重點轉為 B/C/D 的建置，以及與現有課程平台的串接。

---

## 框架概述

```
A  線上課程與影片    → 知識輸入層    ✅ 已完成（獨立平台）
B  任務模組          → 行為引導層    🔨 待開發
C  AI 賦能工具       → 智能洞察層    ✅ 開發中（Vitera Repo）
D  訊息排程系統      → 溝通觸達層    🔨 待開發（D Platform，獨立 Repo）
```

每個元件分為：
- **通用功能**：所有場景都需要，可共用同一套基礎建設
- **場景專屬功能**：各醫療領域的差異化需求
- **MVP 範圍**：第一版必做的最小功能集

## A — 線上課程與影片 ✅ 已完成

### 功能定位

讓使用者在正確的時間點獲得正確的知識，不是一次性的資料夾，而是隨使用者進度動態解鎖的知識流。

> **現況：** 課程平台已完成開發。MVP 階段的工作重點不是「建置 A」，而是「將 B/C/D 與現有課程平台串接」。

---

### 現有平台功能

**內容瀏覽**
- 課程總覽頁：依主題分類、依推薦順序排列
- 搜尋與篩選（依症狀、目標、難度）
- 課程卡片：封面縮圖 + 標題 + 時長 + 完成狀態

**影片播放**
- 獨立網站播放器（瀏覽器開啟，非 LIFF）
- 播放進度記錄（關掉後可從中斷點繼續）
- 播放速度調整（0.75x / 1x / 1.25x）
- 字幕支援

**學習進度**
- 每堂課的完成率（%）
- 整體課程完成進度
- 最近觀看記錄
- 書籤功能（標記重要片段）

**知識測驗（選配）**
- 每個模組結尾附 2-3 題簡單測驗
- 答對才解鎖下一模組
- 錯誤答案提供說明而非直接告知正確答案

**課程解鎖機制**
- 依時間解鎖（加入第 7 天才開放 M2）
- 依任務完成解鎖（完成 10 次打卡才開放進階課程）
- 依訂閱方案解鎖（免費版 / 付費版內容分層）

**推薦系統**
- 根據初始評估結果自動推薦第一堂課
- 根據任務記錄動態推薦相關課程（「你最近睡眠評分偏低，推薦你看這個」）

**後台管理（Admin）**
- 課程上傳（影片 + 封面 + 說明文字）
- 模組排序與解鎖條件設定
- 課程數據：完成率、平均觀看時長、最多跳出點

### MVP 階段：與 B/C/D 的串接需求

A 平台已就緒，MVP 只需要確認以下串接點：

**C → A（評估結果推薦課程）**
- C 完成評估後，依症狀組合帶出對應課程的連結（課程平台網站 URL）
- 不需要修改 A，只需要 C 知道課程 ID 與對應的症狀條件

**B → A（任務完成解鎖課程）**
- 使用者完成特定任務次數後，B 通知 A 解鎖對應課程
- 需確認 A 平台是否提供解鎖 API，或採用共用 DB 欄位判斷

**D → A（訊息帶課程連結）**
- D 推送的訊息需帶入課程頁的網站連結（使用者點擊後跳至獨立課程網站）
- 確認 A 平台各課程的可分享連結格式

## B — 任務模組

### 功能定位

把「知道該做什麼」轉化為「每天真的在做」。任務模組是維持使用者每日回來的核心機制，也是行為改變的引擎。

---

### 通用功能

**任務類型**

| 類型 | 說明 | 適用場景 |
|------|------|---------|
| 勾選打卡 | 簡單的完成 / 未完成 | 換藥、服藥、喝水 |
| 數字輸入 | 輸入具體數值 | 血糖、體重、疼痛指數 |
| 滑桿評分 | 1-10 分拖拉選擇 | 情緒、睡眠品質、疼痛程度 |
| 圖示選擇 | Emoji 或圖片選項 | 滲液顏色、情緒狀態 |
| 照片上傳 | 拍照並上傳 | 傷口記錄、飲食記錄 |
| 計時任務 | 倒數計時 + 引導 | 凱格爾訓練、呼吸練習 |
| 多選打卡 | 選擇今天完成了哪些 | 飲食記錄（吃了哪些食物）|

**每日任務流程**
- 當日任務清單（依優先順序排列）
- 每個任務估計完成時間
- 完成後顯示即時回饋（鼓勵文字 + 動畫）
- 全部完成後顯示每日總結

**連續與進度追蹤**
- 連續打卡天數（Streak）
- 本週完成率（%）
- 30 天完成率趨勢圖
- 最長連續記錄

**彈性補救機制**
- 允許補填昨天未完成的任務（限 24 小時內）
- 中斷後重新開始不清除歷史記錄

**遊戲化機制**
- 里程碑獎章（完成 7 天、30 天、第一次全勤週）
- 解鎖新課程的進度條（可見的激勵）
- 每週成就摘要

**任務排程設計**
- 每個任務可設定推薦完成時段（早上 / 中午 / 晚上）
- 依使用者習慣自動調整（連續 3 天都在晚上做 → 往後調整提醒時間）

**後台管理（Admin）**
- 任務模板庫（各場景預設的任務組合）
- 任務組合綁定方案（訂閱哪個方案 → 解鎖哪些任務）
- 個別使用者任務數據查看

---

### 場景專屬任務設計

**女性健康每日任務組合**

```
更年期方案
├── 症狀記錄（必填）

├── 今日身體感受日記（文字，僅自己可見）
├── 骨盆底肌訓練（同上）

```

**傷口照護每日任務組合**

```
病患端
├── 傷口拍照上傳（照片上傳，必填）
├── 換藥打卡（勾選，並記錄誰換的）

├── 滲液顏色（圖示選擇：清澈/黃/黃綠/帶血）

├── 審閱個案照片

```

**保健品每日任務組合**

```
├── 服用提醒打卡（依品項數量生成對應任務）

└── 飲水量記錄（部分保健品需搭配足夠水分）
```

## C — AI 賦能工具 ✅ 開發中（Vitera Repo）

### 功能定位

C 是整個框架最難複製的護城河。其他三個元件競品都能做，但 AI 工具的品質決定了使用者是否感受到「這個服務真的懂我」。

> **現況：** C 即為 Vitera repo，各醫療場景的 AI 工具以獨立 app 的形式存在（wounds、intimacy、bones、supplements 等）。
>
> **已完成：**
> - `wounds` — 傷口拍照上傳、AI 影像分析（`WoundLog.ai_assessment_summary`）
> - `intimacy` — 性健康評估問卷（`IntimacyAssessment`）
> - `bones` — 足部評估（`FootAssessment`、`FootImage`）
> - `supplements` — 保健品管理
>
> **MVP 待補齊：**
> - 女性健康（更年期）症狀評估問卷與雷達圖結果頁
> - 週期性 AI 分析報告生成（cron + Claude API）
> - 傷口 AI 結果頁的趨勢比較（今天 vs 昨天）
> - 異常偵測邏輯，觸發事件傳至 D Platform

---

### 通用功能

**評估問卷引擎（所有場景的入口）**
- 支援動態分支邏輯（前一題的答案決定下一題）
- 多種題型：單選、多選、滑桿、數字輸入
- 進度條顯示（讓使用者知道還有幾題）
- 中途儲存（可關掉後繼續填）
- 完成後即時生成結果（不要讓使用者等待）

**個人化引擎**
- 根據評估結果自動分配方案（A、B 的內容組合）
- 根據任務完成行為動態調整推薦（完成率高 → 增加難度；完成率低 → 簡化任務）
- 定期重新評估（每 4 週提示重測，追蹤變化）

**進展分析報告（週期性生成）**
- 每 2 週自動生成
- 包含：趨勢圖、關鍵洞察（AI 文字分析）、下一步建議
- 語氣設定：溫暖、具體、不說教
- 可分享（截圖友善設計）

**異常偵測與警示**
- 設定各指標的正常範圍
- 連續 N 天數據異常 → 觸發警示
- 警示分級：關心提醒 / 建議諮詢 / 立即就醫

**AI 問答（聊天介面）**
- Claude API 實作
- System Prompt 設定：同理語氣、有醫療依據、不給診斷、適時建議就醫
- 常見問題快問快答（降低輸入門檻）
- 聊天記錄保存（僅本人可見）
- 每次回答結尾附免責聲明

### AI 工具設計原則

**輸出要讓使用者想截圖分享**
- 視覺化優先（雷達圖、趨勢線、時間軸）
- 個人化細節（包含使用者的實際數據，不是通用文字）
- 標題語氣：「你的更年期報告」而非「評估結果」

**永遠提供下一步**
- AI 報告的最後一頁必定是「你可以做的三件事」
- 下一步要具體可執行（不是「多休息」，是「今晚試試這個 5 分鐘呼吸練習」）

**免費工具作為流量入口**
- 評估問卷 + 即時結果 → 免費開放
- 完整報告 + 歷史追蹤 → 需加入 LINE OA / 付費

## D — 訊息模板

### 功能定位

D 是使用者感受到「有人在陪伴」的介面。好的訊息不像通知，而像一個懂你的朋友在關心你。

---

### 通用功能

**觸發機制類型**

| 觸發類型 | 說明 | 範例 |
|---------|------|------|
| 時間觸發 | 固定時間推送 | 每日 09:00 任務提醒 |
| 事件觸發 | 特定事件發生後 | AI 報告生成完成通知 |
| 行為觸發 | 根據使用者行為 | 連續 3 天未打卡 |
| 進度觸發 | 達到特定里程碑 | 完成 7 天連續打卡 |
| 週期觸發 | 每隔 N 天 | 每 2 週的進展報告通知 |
| 異常觸發 | 數據超出正常範圍 | 傷口感染跡象警示 |

**訊息格式支援**
- 純文字訊息
- 圖文卡片（封面圖 + 標題 + 按鈕）
- 快速回覆按鈕（讓使用者一鍵回應）
- 輪播卡片（多個選項橫向滑動）
- Flex Message（自定義版面）

**個人化變數**
```
{{user_name}}        → 使用者姓名
{{streak_count}}     → 連續打卡天數
{{completion_rate}}  → 本週完成率
{{symptom_summary}}  → 本週症狀摘要
{{task_remaining}}   → 今日剩餘任務數
{{wound_progress}}   → 傷口縮小百分比
{{next_course}}      → 推薦的下一堂課
```

**頻率控制**
- 每位使用者每日最多收到 N 則訊息（避免打擾感）
- 使用者可自訂通知時段（不要在幾點到幾點打擾我）
- 靜音模式（暫停通知 1 天 / 3 天 / 1 週）

**訊息成效分析（後台）**
- 每則訊息的開啟率
- 點擊 CTA 的比率
- 訊息後的行為（有沒有完成任務）
- A/B 測試結果

### 訊息設計原則

**語氣設定**
- 永遠用名字稱呼使用者
- 鼓勵而非督促：「繼續加油」而非「你還沒完成」
- 失去聯繫時關心而非催促
- 特殊場景（性健康）：語氣更輕柔，完全無評判

**CTA 設計**
- 每則訊息最多一個主要 CTA
- 按鈕文字要說動作：「查看報告」而非「點我」
- 盡量讓使用者在 LINE 內完成，不要跳出

## 跨元件整合的關鍵設計點

**C（Vitera）→ A 的連動**
- AI 評估報告的結果直接連結到課程平台的網站 URL（獨立網站，非 LIFF）
- 進展分析發現弱點 → 推薦對應課程
- 課程尚未上架時：連結暫以外部資源（YouTube / 衛教連結）取代，架構不需改動

**B → C（Vitera）的連動**
- 任務打卡數據（B）需寫入 Vitera 後端 DB，供 AI 進展分析使用
- 任務完成率影響 AI 的個人化調整
- 需確認 B 的資料存儲位置（與 Vitera DB 共用或透過 API 同步）

**C（Vitera）→ D（D Platform）的連動**
- Vitera 偵測到異常 → 呼叫 `POST /events` 至 D Platform → D 觸發對應訊息
- Vitera 報告生成完成 → 呼叫 `POST /events { type: "report_ready" }` → D 通知使用者
- C 是 D 最主要的事件來源

**B → D（D Platform）的連動**
- 連續打卡天數達里程碑 → B 呼叫 `POST /events { type: "milestone_reached" }` → D 觸發慶祝訊息
- 超過 N 天未打卡 → D 依自身排程偵測並觸發挽回訊息（不需 B 主動通知）
