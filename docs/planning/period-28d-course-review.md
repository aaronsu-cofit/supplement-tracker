# 28 天生理週期課程 — 設計 Review 待議題

> 提供給團隊（Zoly / Grace / Olga / 宜婷 / Ashlyn / Aaron / 洪啟修）作為下次討論用。
> 內容為審視 Zoly 目前 28 天 spreadsheet（28 天週期、經期 7 天版本）後彙整的待釐清項目。
> 每一條後面附「該決定的問題」，不是答案。

---

## A. 產品框架未定（最重要，會影響所有後續決定）

目前內容像「magazine subscription」— 每天送一篇，看完即逝。需要先選定產品定位：

| 框架 | 核心承諾 | 現有內容契合度 |
|---|---|---|
| **A.** 3 個 keystone 習慣 × 28 天 streak | 28 天後養成 N 個跟週期共處的習慣 | 中（現任務太碎） |
| **B.** Symptom → Pattern → Insight 閉環 | 第二輪起給個人化 report | 低（要重設計） |
| **C.** 產品 bundle 綁定的 28 天計畫 | 配 cofit 保健品 / 工具完成 28 天 | 中（內容已有保健品提示） |
| **D.** 身分轉變課程 | 改變對自己週期的態度 | **高**（口氣已 80% 對位） |

**團隊要決的：選哪一個？目前是哪一個？是否要全部都沾一點（會四不像）？**

---

## B. 「沒在養成習慣」

84 個任務 = 84 個獨一無二的當天 advice，沒有任何行為被重複到能變成習慣。28 天結束後 user 學到了什麼可以延續的東西？

**該決定的**：
- 要不要抽出 2-3 個跨整個 28 天的 keystone habit？（例：紀錄症狀、喝水、伸展）
- 跨多輪 cycle 的話，這幾個 habit 是不是課程的真正 backbone？

---

## C. 「任務的意義」沒有 theory of change

例：
- Day 14「穿上最閃亮的服裝」 → 為什麼？對 user 帶來什麼？
- Day 12「主持會議或演說」 → 大部分人那天根本沒會議
- Day 18「拒絕無效社交」 → user 怎麼判斷哪些社交無效？

很多任務是 performative aesthetic，不是行為設計。

**該決定的**：每一個任務該回答「**做了會有什麼具體改變**」+「**做不到的人替代版是什麼**」。要不要請 Grace 把任務重新審一遍？

---

## D. Persona 沒鎖定，但內容暗示一個

反推內容：上班族 / 25-35 / 有社交圈 / 對打扮有意願 / IG 文化 / 有運動空間 / 有保健品消費習慣。

**該決定的**：
- 行銷端要明確 target persona — 學生、新手媽媽、退休、內向不愛打扮的人**不是這個**
- 不適合的人加進來 → 流失 → 一星 → LINE 封鎖。要先過濾，不要照單全收

---

## E. 主要排除人群佔比可能 ≥ 30%

「典型 28 天 cycle」設計直接排除：
- PCOS / 多囊（25% 育齡女性）
- 更年期前期 / Perimenopause（40+）
- 服用避孕藥 / 含荷爾蒙 IUD
- 產後 / 哺乳中
- 子宮內膜異位（10%）

**該決定的**：
- onboarding 加一題「你的週期是？」(規律 / 不規律 / 服用避孕藥 / 更年期 / 其他)？
- 不規律的人引去**不同**版本（哪怕是「先了解自己週期」入門版）？
- 還是行銷端就明說「適合規律 28 天週期者」？

---

## F. Onboarding 收的資料**沒被用到**

團隊規劃要問「上次經期 / 週期長度 / 經期天數」。但目前腳本寫死，跟使用者填的數值毫無關聯：
- 你週期 26 天 → 系統還是 Day 14 推「排卵期」
- 你經期 5 天 → 系統還是 Day 7 收「經期尾聲」（你早結束了）

**該決定的**：
- 收三個問題只為演戲 = 比不問更糟（騙了 user 期待）
- 要讓資料真的影響推送 schedule，還是乾脆不問？
- 至少把「Day 14 排卵期」改成「**根據你填的週期**，今天大約是排卵期」

---

## G. 28 天腳本是一條軌道，**沒有 fallback**

腳本假設 user：每天乖乖讀、該回應的時候會回應、月經剛好 Day 28→Day 29 來。

真實場景全部沒處理：
- User Day 5 開始忽略訊息 → Day 8 系統繼續推「黃金週」自言自語
- User Day 22 月經早來 → 系統 Day 23 還在「乳房脹」 → 完全脫節
- User 沒回 postback「月經來了嗎?」→ 系統卡住或亂跑
- User Day 29 月經沒來 → 重啟？延後？沉默？

**該決定的**：
- 每幾天插一個 health-check（「妳還在嗎？」）允許重置
- 月經早來 / 晚來都要設計 reset 路徑
- 沒回應的 postback 怎麼處理（timeout fallback 文案是什麼？）

---

## H. Day 29+ 完全沒設計 → 一次性產品

Day 28 結尾「新循環即將啟動」… 然後**？**

| 選項 | 結果 |
|---|---|
| 自動 restart Day 1 | 一模一樣內容 = 第二月就死 |
| 系統停止 | OA 變死，user 放生 |
| 沒人接手 | user 自己決定要不要再開 |

**該決定的**：
- 規劃 cycle 2 / 3 / 4 的差異化策略
- 訂閱化（每月升級一點 / 解鎖個人化 report）還是一次性
- 沒有 cycle 2 設計 = 賺一次走人，要事先講清楚

---

## I. 沒有任何 engagement 訊號被收集

- 哪個 push 用戶有讀？不知道（LINE 不給 read receipt）
- 哪個 task 用戶有做？沒有 postback 設計就沒資料
- 哪幾天 user 完全沒打開？沒測量
- 哪些文案 retention 高 / 低？沒辦法 A/B

**結果**：上線後永遠沒有 data-driven 改進依據。第二版只能憑直覺再猜一次。

**該決定的**：
- 每個 task ☐ 變成 postback 紀錄完成
- LIFF 開啟頻率紀錄
- 「24 小時內有沒有 LINE 回覆」當「讀過 proxy」
- Engagement dashboard（即使簡單表格也好）

---

## J. TCM vs evidence-based 立場混亂

同一份腳本：
- 「紅糖薑茶」、「桂圓紅棗茶」、「補血」、「溫補」 → 中醫脈絡
- 「HIIT」、「Omega-3」、「維生素 D / B 群」 → 西醫 / 營養學
- 「黃金週」、「能量場強大」 → wellness culture

**該決定的**：
- 對內：明確產品立場 「整合東方智慧與現代營養學」之類的 narrative
- 對外（user）：要不要在 onboarding 講清楚？
- 信實證的人會質疑時，客服怎麼回？

---

## K. 隱私 / 資料同意完全沒談（**法務風險**）

User 跟 OA 互動會留下：月經週期、經血量、情緒狀態、用藥、性行為（若加入）、受孕意圖。

這些是**高敏感 PII**。台灣個資法、未來歐盟 GDPR、加州 CCPA 都會看。

- 加入 OA 時有 consent flow 嗎？— 沒看到
- 資料存哪？保留多久？怎麼刪？— 沒設計
- ML 訓練 / 報表 / 第三方分享？— 沒講

**最壞情況**：user 截圖 OA 裡她的紀錄被外洩 → 公關危機 + 法務 + 客訴。

**該決定的**：
- onboarding 第一頁是隱私聲明 + 同意框
- data retention policy
- user 可以刪除自己所有資料的入口
- 法務介入 review

---

## L. 跟 cofit 既有產品 / 內容的連動是 0

cofit 已有：保健品、傷口照護、足部、親密、補給品庫、鄭醫師營養課、cofit pro 會員⋯

Zoly 這份**沒一個 cross-link**：
- Day 8「補抗氧化」→ 沒有 cofit 抗氧化保健品連結
- Day 15「補 Omega-3」→ 沒有 cofit 魚油連結
- Day 24「補 B 群」→ 沒有 cofit B 群連結
- 課程結束後 → 沒有「想學更多 → 鄭醫師營養課」漏斗

**80% 商業價值在這些連動，全部空白。**

**該決定的**：
- 每個 task / 衛教指到的 cofit 商品 / 服務 / 內容（Olga 提的鄭醫師資料庫就是現成材料）
- 課程結束的下一站是哪
- course = 保健品的「使用情境推銷器」，這個定位 OK 嗎

## M. 「告訴使用者該做什麼」vs「教使用者理解原理」

最深的批判：腳本**從來沒教用戶內化原理**，只是發指令。
- 「Day 14 妳是太陽，戰袍穿起來」→ 為什麼？沒講機制
- 「Day 18 黑巧克力」→ 因為什麼成分？沒講
- 「Day 22 寫感謝清單」→ 為什麼黃體期適合感恩？沒講

User 28 天結束後**沒有 model 來理解自己**。下個月狀況改變（壓力大、生病、出差）她不知道怎麼調整。

**該決定的**：
- 每天的 5 個衛教分類（能量／身體／情緒／營養／運動）改成 explanation 不只 advice？
- 讓 user 28 天後能**自己解釋**為什麼今天該做什麼
- 這也直接呼應 A 點的「身分轉變」框架
