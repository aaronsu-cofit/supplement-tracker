# DTx 整合架構：A+B+C+D 與所有模組的對應關係

> **版本：** v1.0
> **日期：** 2026-03-11

---

## 一、核心概念：C 是整個框架的大腦

A+B+C+D 不是四個平行的模組，而是有依賴關係的系統：

```
                    使用者
                      │
              LINE OA / LIFF
                      │
        ┌─────────────┼─────────────┐
        ↓             ↓             ↓
    [A 課程]      [B 任務]      [D 訊息]
        ↑             ↑             ↑
        └─────────────┼─────────────┘
                      │
               ┌──[C AI工具]──┐
               │              │
          [評估引擎]      [分析引擎]
          ↙      ↘        ↙      ↘
      分配A     分配B   觸發D   產出報告
```

**C 做三件事：**
1. **輸入端**：收集使用者數據（評估問卷 + 任務打卡數據 + 健康數據）
2. **處理端**：分析、判斷、個人化
3. **輸出端**：決定推送哪些 A、分配哪些 B、觸發哪些 D

---

## 二、A+B+C+D 四個元件的職責邊界

| 元件 | 職責 | 不負責 |
|------|------|--------|
| **A 課程** | 內容儲存與播放 | 決定誰該看哪個課程（這是 C 的事）|
| **B 任務** | 任務呈現與記錄 | 決定要分配哪些任務（這是 C 的事）|
| **C AI工具** | 評估、分析、個人化、異常偵測 | 呈現內容與互動（這是 A/B/D 的事）|
| **D 訊息** | 訊息推送與觸達 | 決定何時推什麼（觸發邏輯由 C 驅動）|

---

## 三、C 如何驅動 A、B、D

### C → A（決定推薦什麼課程）

```
評估結果：熱潮紅嚴重 + 睡眠差
    ↓
C 判斷：優先推 M2（飲食）和 M4（睡眠）
    ↓
A 顯示：課程列表排序調整，M2/M4 置頂
```

### C → B（決定分配什麼任務）

```
評估結果：骨盆底肌功能弱
    ↓
C 判斷：每日必做凱格爾訓練，症狀記錄加入「漏尿頻率」
    ↓
B 顯示：今日任務包含凱格爾計時器

14 天後 B 的數據回饋給 C：
任務完成率 80% → C 判斷可升難度
任務完成率 20% → C 判斷要簡化任務
```

### C → D（決定何時推什麼訊息）

```
C 偵測到：連續 3 天睡眠評分 < 2
    ↓
D 觸發：推送睡眠課程推薦 + 關心訊息

C 偵測到：傷口照片邊緣異常
    ↓
D 觸發：推送警示給病患 + 通知護理師
```

## 五、完整模組對應表

| 模組 | 插入位置 | 說明 |
|------|---------|------|
| **健康數據整合** | → C 輸入 | 穿戴裝置/血糖機數據成為 C 分析的原料 |
| **心理支持/正念** | → A + B | 新增一種課程類型(A) + 一種任務類型(B) |
| **照護者模式** | → B + D | 照護者看到的是鏡像任務清單(B) + 收到同步通知(D) |
| **社群支持** | → B + D | 群組挑戰是特殊任務(B) + 群組通知走 D 的管道 |
| **人力諮詢層** | → C 延伸 | C 判斷需要真人時，升級到人力諮詢（C 的 fallback）|
| **OMO 銜接** | → C + D | C 打包健康摘要，D 發送就診提醒 |
| **商城/保健品** | → C + D | C 根據症狀推薦商品，D 發送服用提醒與回購通知 |
| **成就系統** | → B 附加層 | 任務完成後的獎勵邏輯，掛在 B 上面 |
| **進度可視化** | → C 輸出 | C 分析結果的另一種呈現形式 |

## 六、兩個場景的完整功能架構

### 女性健康

```
C 評估引擎
└── 更年期症狀問卷（10題）
└── 性健康評估（FSFI 精簡版）
└── 輸出：症狀雷達圖 + 方案分流
          │
          ├──→ A 課程
          │    ├── M1 認識更年期
          │    ├── M3 凱格爾教學（依評估推薦）
          │    ├── M4 睡眠修復（依睡眠評分推薦）
          │    └── 正念音頻（心理支持模組 → A）
          │
          ├──→ B 任務（每日）
          │    ├── 症狀記錄（熱潮紅/睡眠/情緒）
          │    ├── 凱格爾計時器（心理支持模組 → B）
          │    └── 睡前呼吸練習（選做）
          │
          ├──→ C 分析引擎（每 2 週）
          │    ├── 症狀趨勢分析
          │    ├── 洞察：「熱潮紅與週三壓力有關」
          │    └── → 觸發 D 推送報告通知
          │
          ├──→ D 訊息
          │    ├── 每日任務提醒（時間觸發）
          │    ├── 週洞察訊息（C 分析完成觸發）
          │    ├── 里程碑慶祝（B 達成觸發）
          │    └── 付費轉換（D14 週期觸發）
          │
          └──→ 延伸模組（Phase 2+）
               ├── 商城：保健品推薦（C 推薦 → D 通知）
               ├── 人力諮詢：症狀持續 → 一對一預約
               └── OMO：就醫前健康摘要打包
```

### 傷口照護

```
C 評估引擎
└── 設定問卷（傷口類型/換藥頻率/護理師）
└── 輸出：照護方案 + 換藥提醒排程
          │
          ├──→ A 課程
          │    ├── WC-A2 換藥 SOP（依傷口類型推薦）
          │    ├── WC-A3 感染警訊識別
          │    └── WC-A5 糖尿病足（依傷口類型條件顯示）
          │
          ├──→ B 任務（每日）
          │    ├── 傷口拍照上傳（照片上傳任務）
          │    ├── 換藥打卡
          │    ├── 疼痛 / 滲液記錄（滑桿 + 圖示）
          │    └── 照護者任務（照護者模式 → B）
          │
          ├──→ C AI 工具
          │    ├── 影像分析（面積/感染/癒合階段）
          │    ├── 異常偵測（連 2 天惡化 → 觸發警示）
          │    ├── 癒合時間軸（進度可視化）
          │    └── 護理師評估初稿自動生成（Phase 2）
          │
          ├──→ D 訊息
          │    ├── 每日拍照提醒（時間觸發）
          │    ├── AI 分析完成通知（事件觸發）
          │    ├── 異常警示（C 異常偵測觸發）→ 病患 + 護理師
          │    └── 照護者同步通知（照護者模式 → D）
          │
          └──→ 延伸模組（Phase 2+）
               ├── 護理師後台（批次管理、週報生成）
               ├── OMO：AI 前導病歷打包 → 診所轉介
               └── 商城：傷口敷料推薦（C 推薦 → D 通知）
```

## 八、技術層對應

| 業務模組 | 技術實作位置 |
|---------|------------|
| A 課程播放 | LIFF 頁面（/courses, /player）|
| B 任務引擎 | LIFF 頁面（/tasks）+ 後端任務狀態管理 |
| C 評估引擎 | LIFF 頁面（/onboarding）+ 後端問卷邏輯 |
| C 分析引擎 | 後端排程任務（每 2 週執行）+ Claude API |
| C 影像分析 | 後端 API（接影像 AI 服務）|
| C 異常偵測 | 後端規則引擎（每日任務完成後觸發）|
| D 訊息推送 | LINE Messaging API + 後端排程 |
| D 觸發邏輯 | 後端事件系統（任務完成/分析完成/異常偵測 → 推送）|
| 健康數據整合 | 後端 API（Apple Health / Google Fit OAuth）|
| 人力諮詢 | 第三方視訊 API（Whereby / 自建）|
| 商城整合 | 串接既有保健品商城 API |
