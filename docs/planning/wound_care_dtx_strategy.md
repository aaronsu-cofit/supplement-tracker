# 傷口照護 DTx 產品策略與執行規劃

> **版本：** v1.0
> **日期：** 2026-03-11
> **載體：** LINE Official Account + LIFF
> **定位：** AI 驅動的傷口追蹤與護理輔助平台

---

## 一、背景與策略邏輯

### 與其他 DTx 領域的根本差異

傷口照護的 A+B+C+D 框架重心與其他領域不同：

| 領域 | 框架核心 | 沒有 C 的話 |
|------|---------|-----------|
| 女性健康 / 減脂 | A+B（知識與任務）驅動使用者持續參與 | 仍有課程與陪跑價值 |
| **傷口照護** | **C（AI 影像分析）是唯一差異化** | 就只是普通的提醒 App |

因此，傷口照護的策略邏輯是：**先有強大的 AI 工具，再圍繞它設計 A、B、D。**

### 為什麼值得做

| 評估維度 | 說明 |
|---------|------|
| 醫療人力缺口 | 護理師花大量時間在傷口拍照、記錄、追蹤，重複性高、可自動化 |
| AI 技術壁壘 | 傷口影像分析是真實的技術護城河，不容易被複製 |
| Vitera 既有基礎 | wounds app 已存在，技術積累可直接延伸 |
| 市場集中 | 目標客戶是機構而非散客，一次成交影響力大 |
| 慢性傷口長期性 | 糖尿病足、壓瘡反覆發作，有持續互動與付費基礎 |

## 二、目標對象與服務模式

### 核心策略：B2B 優先，B2C 輔助

傷口照護的付費能力與規模都在機構端，病患端作為使用者體驗的一部分，不是主要營收來源。

```
主要付費客戶（B2B）
├── 居家護理所（每位個案護理師管理成本高）
├── 洗腎中心（糖尿病足是最常見的共病）
├── 長照機構（壓瘡是最大照護負擔之一）
└── 外科 / 整形外科診所（術後傷口追蹤）

使用者（B2C，由機構導入）
├── 居家傷口病患
├── 居家照護者（家屬）
└── 糖尿病患者（預防性管理）
```

### 為什麼不從 B2C 切入

- 傷口病患通常是被動的，不會主動尋找數位工具
- 沒有診所或護理所協助導入，很難找到使用者
- 護理師才是最有動機使用 AI 工具的人（因為可以節省他們的時間）

## 三、聚焦的傷口類型

不要做全科，優先聚焦慢性傷口：

| 傷口類型 | 理由 | 主要對應機構 |
|---------|------|------------|
| **糖尿病足潰瘍** | 反覆發作、長期追蹤、AI 偵測早期惡化價值高 | 洗腎中心、新陳代謝科 |
| **壓瘡（褥瘡）** | 長照機構最大照護負擔、預防與追蹤需求強 | 長照機構、居家護理所 |
| **術後傷口** | 量大但週期短，適合作為初期量的來源 | 外科診所、整形外科 |
| **靜脈性下肢潰瘍** | 老年人常見，反覆發作 | 居家護理所 |

**建議第一階段聚焦：糖尿病足 + 壓瘡**
理由：慢性反覆、護理需求強、機構付費意願高。

## 四、A+B+C+D 框架詳細設計

### A — 線上課程與影片

主要服務對象是**病患與居家照護者**，讓他們具備基本知識，減少護理師的衛教時間。

| 代號 | 主題 | 適用對象 |
|------|------|---------|
| W1 | 傷口類型辨識：你的傷口是哪一種？ | 所有人 |
| W2 | 正確換藥 SOP：手把手圖文教學 | 居家照護者 |
| W3 | 感染警訊：這 5 種狀況要立刻就醫 | 所有人 |
| W4 | 傷口癒合的營養學：蛋白質、維生素 C、鋅怎麼補 | 術後病患 |
| W5 | 糖尿病足預防與日常足部照護 | 糖尿病患者 |
| W6 | 壓瘡預防：翻身頻率、體位、皮膚照護 | 長照環境 |
| W7 | 傷口敷料選擇指南（給照護者看）| 居家照護者 |

**推送方式：** LINE 訊息卡片預覽 → 點擊開 LIFF 播放
**格式：** 3-5 分鐘短影片 + 關鍵步驟圖文（兼顧長輩使用習慣）

---

### B — 任務模組

**病患 / 照護者每日任務：**

```
✅ 傷口每日拍照上傳（核心任務）
   - LIFF 相機介面，自動上傳並觸發 AI 分析

✅ 換藥打卡
   - 今天換藥了嗎？自己換 / 護理師換 / 未換（填原因）

✅ 傷口狀態快速記錄
   - 疼痛程度（1-10 分滑桿）
   - 滲液顏色（清澈 / 黃 / 黃綠 / 帶血，圖示選擇）
   - 周邊皮膚狀況（正常 / 發紅 / 腫脹）

✅ 血糖記錄（糖尿病足患者）

✅ 蛋白質攝取打卡
   - 今天有沒有吃足夠的蛋白質？（加速癒合提示）
```

**護理師後台每日任務（B2B）：**

```
✅ 今日待審閱個案清單
   - AI 自動將「有異常跡象」的個案排在最前面

✅ 遠端傷口照片審閱與批准
   - 確認 AI 分析結果是否正確，一鍵批准或修改

✅ 自動生成週報表確認後送出
```

### C — AI 賦能工具（核心差異化）

**工具 1：AI 傷口影像分析**

病患每天上傳傷口照片，AI 自動輸出：

| 分析項目 | 說明 |
|---------|------|
| 傷口面積估算 | 與上次比較，顯示縮小/擴大百分比 |
| 邊緣狀態 | 清晰癒合中 / 邊緣不規則需注意 |
| 感染跡象偵測 | 發紅範圍、腫脹、滲液顏色異常 |
| 癒合階段分類 | 發炎期 / 增生期 / 重塑期 |
| 組織狀態 | 健康肉芽組織 / 壞死組織比例（進階）|

輸出給病患：
```
📊 今日傷口分析

面積：較昨日縮小 8% ✅
邊緣：癒合正常
滲液：少量清澈（正常範圍）
整體評估：持續進展良好，繼續保持！

下次換藥建議：明天
```

輸出給護理師：
```
⚠️ 個案 #042 林○○
傷口邊緣今日出現不規則擴展，
滲液顏色由清澈轉為黃色。
建議本週安排回診評估。
```

---

**工具 2：護理師工作流程自動化**

> 解決護理人力不足的核心工具

- AI 根據照片自動產出**傷口評估初稿**（護理師只需確認或微調）
- 自動計算傷口面積並填入標準病歷格式（PUSH、BWAT 等評估量表）
- **批次管理介面**：一次查看所有個案今日狀況，異常優先排列
- 匯出報表（週 / 月），供機構存檔或健保申報

預估效益：每位個案每次記錄從 10-15 分鐘縮短至 2-3 分鐘。

---

**工具 3：風險警示系統**

| 觸發條件 | 警示對象 | 動作 |
|---------|---------|------|
| AI 偵測傷口惡化（連續 2 天）| 護理師 | LINE 推送警示 + 建議介入時機 |
| 感染跡象評分超過閾值 | 護理師 + 病患 | 即時警示 + 建議就醫 |
| 連續 3 天未上傳照片 | 護理師 | 提醒跟進確認個案狀況 |
| 血糖異常（糖尿病足）| 護理師 | 同步警示（雙重風險提醒）|

---

**工具 4：癒合進展時間軸**

- 視覺化呈現傷口從第 1 天到今天的縮時影像
- 自動標記關鍵節點（換藥方式改變、感染發生、開始好轉）
- 讓病患看見「有在進步」，提升配合度與黏著度
- 護理師可直接下載時間軸報告用於交班或轉介

---

### D — 訊息模板

**病患 / 照護者端：**

| 時間 | 類型 | 訊息範例 |
|------|------|---------|
| 每日 09:00 | 拍照提醒 | 「早安 📸 記得拍今天的傷口照片，讓我們幫你追蹤癒合進度 → [立即上傳]」 |
| 換藥時間（依設定）| 換藥提醒 | 「今天是換藥日 💊 換完記得打卡，有問題可以問我」 |
| AI 分析完成後 | 分析報告通知 | 「傷口分析報告出來了！這週縮小了 15%，繼續加油 → [查看報告]」 |
| 異常偵測 | 警示 | 「我們注意到一些需要關注的狀況，建議今天聯絡您的護理師 ⚠️ → [查看詳情]」 |
| 連續 7 天未改善 | 關心 | 「這週的進展需要多一點關注，幫你預約護理師電話評估好嗎？」 |
| 達成里程碑 | 慶祝 | 「🎉 太棒了！傷口面積縮小超過 50%，你做到了！」 |

**護理師端（B2B）：**

| 觸發條件 | 訊息 |
|---------|------|
| 個案傷口異常 | 「個案 [名字] #042 今日傷口 AI 評估出現異常，建議本週跟進 → [查看]」 |
| 個案 3 天未上傳 | 「個案 [名字] 已 3 天未傳照片，請確認狀況」 |
| 週報自動生成 | 「本週 [N] 位個案報告已自動生成，請確認後送出 → [審閱週報]」 |
| 月結報表 | 「[月份] 傷口照護月報已完成，含 [N] 位個案的完整紀錄 → [下載報表]」 |

## 五、使用者完整旅程

### 病患端旅程

```
[由護理師或診所導入，加入 LINE OA]
            ↓
歡迎訊息 + 基本資料設定（LIFF）
（傷口類型、換藥頻率、主治護理師）
            ↓
第一次拍照上傳 → AI 建立基準線
            ↓
每日 LINE 任務提醒 → 拍照 + 記錄
            ↓
每次分析完成 → 收到報告通知
            ↓
每週收到「癒合進展週報」
            ↓
傷口癒合完成 → 結案報告
（急性傷口：2-8 週 / 慢性傷口：持續管理）
```

### 護理師端旅程（B2B）

```
[機構採購，護理師帳號開通]
            ↓
建立個案清單（匯入或手動新增）
            ↓
傳送 LINE OA 邀請給每位個案
            ↓
每日開啟後台儀表板
→ 查看所有個案今日照片 + AI 分析結果
→ AI 異常個案排在最前面
→ 確認分析報告（批准或修改）
            ↓
每週自動生成個案週報，一鍵送出
            ↓
月底匯出月結報表（健保/機構存檔用）
```

## 七、商業模式

### 定價結構（B2B 為主）

| 方案 | 對象 | 內容 | 建議定價 |
|------|------|------|---------|
| 機構方案 | 居家護理所 / 長照機構 | 護理師後台 + 無限個案數 + AI 分析 + 自動報表 | NT$8,000-15,000/月 |
| 診所方案 | 外科 / 整形外科診所 | 護理師後台 + 個案數上限 + AI 分析 | NT$3,000-6,000/月 |
| 個案加購 | 機構超出上限 | 每位個案追加費用 | NT$200-400/人/月 |
| 病患自費（B2C）| 個人 | 拍照上傳 + AI 分析 + 課程 | NT$199-299/月 |

### 為什麼不用 per-use 計費

機構需要可預測的月費支出，且護理師使用頻率高，訂閱制對雙方都更簡單。

### 保健品整合機會

- 傷口癒合後期推薦「膠原蛋白 + 維生素 C + 鋅」補充包
- 在課程 W4 和分析報告中自然置入
- 結合保健品訂閱（每月配送）

## 八、MVP 執行計畫

### Phase 1：核心工具驗證（6-8 週開發）

目標：驗證護理師願意用 AI 工具取代手動記錄

**必做功能：**
- [ ] 病患側 LIFF 拍照上傳介面
- [ ] AI 傷口基礎分析（面積估算 + 感染跡象偵測）
- [ ] 護理師後台個案清單 + 照片審閱介面
- [ ] 每日拍照提醒 LINE 推送（D）
- [ ] 異常警示推送
- [ ] 基礎換藥打卡任務（B）

**成功指標：**
- 護理師每個案記錄時間縮短 > 50%
- 病患每日照片上傳率 > 60%
- Pilot 機構護理師滿意度 > 4/5

---

### Phase 2：完整服務深化（Phase 1 驗證後）

**必做功能：**
- [ ] 癒合進展時間軸（病患端）
- [ ] 自動週報 / 月報生成（護理師端）
- [ ] 完整課程影片模組（W1-W7）
- [ ] 血糖 + 症狀完整記錄（B 完整版）
- [ ] 付費機制串接

### Phase 3：規模化（有 3+ 家機構付費後）

**必做功能：**

- [ ] 進階 AI 分析（壞死組織比例、癒合速度預測）
- [ ] 轉介通道（AI 判斷需要升級醫療時）

## 九、Pilot 策略

### 如何找到第一家機構

沒有既有診所關係，建議用以下方式切入：

1. **從洗腎中心開始**：台灣洗腎中心密度高，糖尿病足是他們的日常，且洗腎中心規模適中（容易決策），護理人力壓力極大
2. **用免費 Pilot 換數據**：提供 1-2 個月免費試用，換取「節省工時」的真實數據
3. **數據換口碑**：拿到「每個案節省 X 分鐘」的數據後，用這個數字去 pitch 下一家

### Pilot 成功的定義

```
量化指標：
  每位個案護理記錄時間縮短 > 50%
  AI 分析準確率（護理師確認不需修改的比例）> 80%
  個案照片上傳率 > 60%

質化指標：
  護理師表示「願意持續使用」
  機構主管表示「願意付費繼續」
```

## 十一、待討論與決策事項

| 問題 | 選項 | 建議 |
|------|------|------|
| AI 傷口分析用自建模型還是第三方 API？ | 自建（高成本）/ 串接現有醫療 AI 服務 | 初期先用第三方，驗證需求後再評估自建 |
| 護理師後台是 LIFF 還是獨立 Web？ | LIFF（簡單）/ 獨立 Web（功能完整）| 建議 Phase 1 用 LIFF，Phase 2 升級 Web |
| 第一個 Pilot 機構如何找？ | 透過現有人脈 / 主動 cold outreach | 先盤點內部有無洗腎中心或長照相關人脈 |
| 傷口照片的資安與隱私規範？ | 需符合個資法 + 醫療資料規範 | 早期就要確認儲存與傳輸的加密規格 |
| B2C 病患方案要不要做？ | 有 / 沒有 | 建議 Phase 1 先不做，專注 B2B 驗證 |
