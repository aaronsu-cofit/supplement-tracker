28 天生理週期課程
設計 Review Zoly 28-day spreadsheet 讀完後彙整 13 個待議題，給團隊下次討論用。
每一條後面附「該決定的」，不是答案。

tl;dr

一句話總結

目前的 28 天課程是「優秀的內容執筆 + 落後的產品設計」。
Zoly 的文字夠好，但這個產品骨架（targeting / personalization / retention / measurement / commerce）幾乎沒搭。
先決定產品框架，再回頭重切內容，比直接灌進系統有意義。

本份 review 不否定內容創作品質，焦點在「產品設計層」要先補的東西。

核心批判

三件事，一句話三個 NO

① 沒在養成習慣

84 個 task 全是當天 only 的 micro-action。28 天結束 user 學到什麼可以延續的？— 啥也沒有。

② 任務沒 theory of change

「Day 14 穿閃亮服裝」「Day 12 主持會議」 — 大多是 performative aesthetic，不是行為設計。

③ Hook 只到隔天

「明日預告」是 LINE-native，但 Day 28 之後沒下文。
對 user-to-tomorrow 有效，對產品-to-下一站無效。

這三件事連成一句話：這是 magazine subscription，不是 course。

A

產品框架未定（最重要）

目前像 magazine subscription — 每天一篇看完即逝。需要先選定產品定位：

核心承諾現有內容契合度
A. 3 個 keystone 習慣 × 28 天 streak28 天後養成 N 個跟週期共處的習慣中
B. Symptom → Pattern → Insight第二輪起給個人化 report低
C. 產品 bundle 綁定 28 天計畫配 cofit 保健品/工具完成 28 天中
D. 身分轉變課程改變對自己週期的態度高（口氣已 80% 對位）
  • 選哪一個？目前是哪一個？
  • 是否要全部都沾一點？（會四不像）
B

沒在養成習慣 + 任務沒意義

B. 任務不重複

  • Day 1 喝薑茶、Day 2 補鐵、Day 3 喝桂圓紅棗茶
  • 每天不同的飲品 / 動作 / 主題
  • 真正每天反覆的行為 = 0

C. 任務沒講為什麼

  • 「Day 14 穿閃亮服裝」→ 為什麼？
  • 「Day 12 主持會議」→ 大部分人那天根本沒會議
  • 「Day 18 拒絕無效社交」→ 怎麼判斷哪些無效？
  • 抽出 2-3 個跨整個 28 天的 keystone habit？（例：紀錄症狀、喝水、伸展）
  • 每個任務該回答「做了會有什麼具體改變」+「做不到的人替代版是什麼」
  • 請 Grace 把任務重新審一遍？
D·E

Persona 沒鎖定 + 排除 30%+ 人群

D. 內容暗示一個 persona 但沒講

反推：上班族 / 25-35 / 有社交圈 / 對打扮有意願 / IG 文化 / 有運動空間 / 有保健品消費習慣

學生、新手媽媽、退休、內向、不愛打扮的人 — 不是這個。

E. 直接排除 30%+ 育齡女性

  • PCOS / 多囊（25%）
  • 更年期前期（40+）
  • 服用避孕藥 / 含荷爾蒙 IUD
  • 產後 / 哺乳中
  • 子宮內膜異位（10%）
  • onboarding 加一題「你的週期是？」(規律 / 不規律 / 服用避孕藥 / 更年期 / 其他)
  • 不規律的人引去不同版本，還是行銷端就明說「適合規律 28 天週期者」？
F

Onboarding 收的資料沒用到

團隊規劃 onboarding 要問：上次經期 週期長度 經期天數。

但目前腳本完全寫死，跟使用者填的數值毫無關聯：

你週期 26 天

系統還是 Day 14 推「排卵期」

你經期 5 天

系統還是 Day 7 收「經期尾聲」（你早結束了）

收三個問題只為演戲 = 比不問更糟（騙了 user 期待）
  • 要讓資料真的影響推送 schedule，還是乾脆不問？
  • 至少把「Day 14 排卵期」改成「根據你填的週期，今天大約是排卵期」
G

28 天腳本是一條軌道，沒有 fallback

腳本假設 user：每天乖乖讀、該回應的時候會回應、月經剛好 Day 28→Day 29 來。

真實場景全部沒處理：

  • User Day 5 開始忽略訊息 → Day 8 系統繼續推「黃金週」自言自語
  • User Day 22 月經早來 → 系統 Day 23 還在「乳房脹」 → 完全脫節
  • User 沒回 postback「月經來了嗎?」→ 系統卡住或亂跑
  • User Day 29 月經沒來 → 重啟？延後？沉默？
  • 每幾天插一個 health-check（「妳還在嗎？」）允許重置
  • 月經早來 / 晚來都要設計 reset 路徑
  • 沒回應的 postback 怎麼處理（timeout fallback 文案是什麼？）
H

Day 29+ 完全沒設計 → 一次性產品

Day 28 結尾：「新循環即將啟動」… 然後？

選項結果
自動 restart Day 1一模一樣內容 = 第二月就死
系統停止OA 變死，user 放生
沒人接手user 自己決定要不要再開

Apple Health / 一般 period app 大家會留幾年，因為每月有意義。
這個課程目前設計只活一個月。

  • 規劃 cycle 2 / 3 / 4 的差異化策略
  • 訂閱化（每月升級一點 / 解鎖個人化 report）還是一次性？
  • 沒有 cycle 2 設計 = 賺一次走人，要事先講清楚
I

沒有任何 engagement 訊號被收集

  • 哪個 push 用戶有讀？不知道（LINE 不給 read receipt）
  • 哪個 task 用戶有做？沒有 postback 設計就沒資料
  • 哪幾天 user 完全沒打開？沒測量
  • 哪些文案 retention 高 / 低？沒辦法 A/B
上線後永遠沒有 data-driven 改進依據。
第二版只能憑直覺再猜一次。
  • 每個 task ☐ 變成 postback 紀錄完成
  • LIFF 開啟頻率紀錄
  • 「24 小時內有沒有 LINE 回覆」當「讀過 proxy」
  • Engagement dashboard（即使簡單表格也好）
J·K

立場混亂 + 隱私 / 法務空白

J. TCM vs evidence-based 混搭

  • 紅糖薑茶 / 桂圓紅棗茶 / 補血 / 溫補 → 中醫
  • HIIT / Omega-3 / 維生素 D B 群 → 西醫
  • 「黃金週」「能量場強大」 → wellness culture

沒被宣稱也沒被解釋 → 用戶質疑時客服沒得回

K. 隱私同意完全沒談

User 留下：月經週期、經血量、情緒、用藥、性行為、受孕意圖 — 高敏感 PII。

  • consent flow？沒有
  • 資料存哪 / 留多久 / 怎麼刪？沒設計
  • ML / 報表 / 第三方分享？沒講
  • 產品立場明文化（「整合東方智慧與現代營養學」）
  • onboarding 第一頁是隱私聲明 + 同意框
  • data retention policy + user 刪除自己資料的入口
  • 法務介入 review（個資法 / GDPR / CCPA）
L

跟 cofit 既有產品 / 內容的連動是 0

cofit 已有：保健品、傷口照護、足部、親密、補給品庫、鄭醫師營養課、cofit pro 會員⋯

Zoly 這份沒一個 cross-link：

  • Day 8「補抗氧化」→ 沒有 cofit 抗氧化保健品連結
  • Day 15「補 Omega-3」→ 沒有 cofit 魚油連結
  • Day 24「補 B 群」→ 沒有 cofit B 群連結
  • 課程結束後 → 沒有「想學更多 → 鄭醫師營養課」漏斗
80% 商業價值在這些連動，全部空白。
  • 每個 task / 衛教指到的 cofit 商品 / 服務 / 內容（鄭醫師資料庫就是現成材料）
  • 課程結束的下一站是哪
  • course = 保健品的「使用情境推銷器」 — 這個定位 OK 嗎？
M

「告訴使用者該做什麼」vs「教使用者理解原理」

最深的批判：腳本從來沒教用戶內化原理，只是發指令。

  • 「Day 14 妳是太陽，戰袍穿起來」→ 為什麼？沒講機制
  • 「Day 18 黑巧克力」→ 因為什麼成分？沒講
  • 「Day 22 寫感謝清單」→ 為什麼黃體期適合感恩？沒講
User 28 天結束後沒有 model 來理解自己。
下個月狀況改變（壓力大、生病、出差）她不知道怎麼調整。
  • 每天的 5 個衛教分類（能量 / 身體 / 情緒 / 營養 / 運動）改成 explanation 不只 advice？
  • 讓 user 28 天後能自己解釋為什麼今天該做什麼
  • 這也直接呼應 A 點的「身分轉變」框架
