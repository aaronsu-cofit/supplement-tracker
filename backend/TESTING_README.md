## 🛠️ 測試腳本

### 主測試腳本

#### `scripts/test-fastify-complete.sh`
完整的測試套件，包含：
- TypeScript 編譯檢查
- 單元測試執行
- 服務器啟動測試
- 健康檢查
- API 端點驗證
- 性能基準測試

**使用方法：**
```bash
bash scripts/test-fastify-complete.sh
```

### 端點驗證腳本

#### `scripts/verify-endpoints.sh`
驗證所有關鍵 API 端點：
- 健康檢查端點
- 認證端點
- 補充品端點
- 傷口端點
- HQ 管理端點
- 其他端點

**使用方法：**
```bash
bash scripts/verify-endpoints.sh
```

### 性能測試腳本

#### `scripts/benchmark-performance.sh`
使用 wrk 進行性能基準測試：
- 健康檢查端點性能
- 模組列表端點性能
- 產品列表端點性能

**使用方法：**
```bash
# 需要先安裝 wrk
brew install wrk

# 執行測試
bash scripts/benchmark-performance.sh
```

## 🚀 快速開始

### 方案 A: 一鍵完整測試（推薦）

```bash
# 進入後端目錄
cd /Users/chingchingyeh/cofit/dtx-space/Vitera/backend

# 執行完整測試套件
bash scripts/test-fastify-complete.sh
```

### 方案 B: 分步執行

```bash
# 1. 編譯檢查
pnpm run build

# 2. 單元測試
pnpm test

# 3. 啟動服務器（背景執行）
node dist/fastify-app.js &
FASTIFY_PID=$!

# 4. 健康檢查
curl http://localhost:8081/health

# 5. 端點驗證
bash scripts/verify-endpoints.sh

# 6. 性能測試（可選）
bash scripts/benchmark-performance.sh

# 7. 清理
kill $FASTIFY_PID
```

## 🐛 常見問題

### Q1: 測試腳本顯示 "permission denied"
```bash
# 添加執行權限
chmod +x scripts/*.sh
```

### Q2: 服務器無法啟動
```bash
# 檢查端口是否被佔用
lsof -i :8081

# 終止佔用的進程
kill -9 <PID>
```

### Q3: 端點驗證失敗
```bash
# 確認服務器已啟動
curl http://localhost:8081/health

# 查看服務器日誌
node dist/fastify-app.js
```

### Q4: wrk 工具未安裝
```bash
# macOS
brew install wrk

# 或使用替代工具
brew install ab
npm install -g autocannon
```

更多問題請參考 [故障排除指南](TROUBLESHOOTING_GUIDE.md)
